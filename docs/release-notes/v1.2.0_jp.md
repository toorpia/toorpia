# toorPIA Python Client v1.2.0 リリースノート

**リリース日:** 2026-07-07

## 主な変更 (v1.1.2 以降)

### 新機能 (feat)

- **embedding データ系統に対応: `basemap_embedding()` / `addplot_embedding()` (#33)**
  - 埋め込みベクトル（行=サンプル、列=次元の CSV。ヘッダ任意、先頭の非数値 ID 列は自動検出）からのベースマップ作成と、そのマップへの追加プロット（異常判定付き）に対応しました。
  - CSV ファイルパス（str / list）に加え、**2次元 `numpy.ndarray` / `pandas.DataFrame` の直接入力**に対応しました。ndarray はヘッダなし CSV、DataFrame はヘッダ付き CSV に自動変換して送信します（ヘッダなしにすることで、サーバが生成する列名がベースマップと追加プロットの間で一致します）。
  - `basemap_embedding()` のオプションは `l2_normalization`（既定 True。ノルムが情報を持つ埋め込みでは False を指定）と `id_columns`（省略時は自動検出）の2つです。未指定のパラメータは送信せず、サーバ既定に委ねます。
  - `addplot_embedding()` は前処理オプションを受け付けません。サーバ側でベースマップから自動継承されるため、ベースマップと追加プロットの前処理の同一性が保証されます。
  - `vector_normalization` は embedding では無効（エンジンが常にユークリッド距離モードを使用）のため、引数自体を設けていません。
  - 利用には `/data/basemap_embedding`, `/data/addplot_embedding` エンドポイントを持つバージョンの toorPIA backend が必要です。
  - なお、embedding マップでは特徴量分析（`get_addplot_features()`）はサポートされません。

### 修正 (fix)

- **`samples/test_addplot_history.py` が必ず中断する問題を修正 (#33)**
  - `client.addplot()` の戻り値仕様変更（ndarray → dict）にサンプルが追従しておらず、`add_result.shape` の AttributeError でテストが常に中断していました。`add_result['xyData'].shape` に修正し、全章の通し実行を確認済みです。

### ドキュメント / サンプル

- `docs/api-reference.md`: 両メソッドのリファレンス、Process Method Compatibility マトリクスへの embedding 行/列追加、前処理パラメータ詳細
- `README.md`: Embedding Data Analysis セクション（CSV / ndarray 両方の例）

## 後方互換性

- 既存メソッドへの変更はありません（`toorpia/client.py` の差分は追加のみ）。v1.1.x からの移行に伴うユーザコードの修正は不要です。
- 既存の csvform / waveform / DataFrame 系統の動作・API 仕様に変更はありません。
